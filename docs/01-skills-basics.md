# Skills 基础概念

## 为什么需要 Skills 和 Rules

### 问题背景

在使用 Claude Code 进行开发时，我们经常遇到：
- **重复工作** - 每次都要重复说明相同的偏好和规范
- **上下文丢失** - 切换会话后之前的约定全部丢失
- **一致性问题** - 不同时间得到的输出格式/风格不一致
- **效率低下** - 大量时间花在"教育" AI 而非解决问题

### Skills 解决什么

**Skills** 是可复用的能力模块，让 Claude Code 能够：
- 执行特定领域的专业任务
- 遵循固定的执行流程
- 输出标准化的结果格式

### Rules 解决什么

**Rules** 是行为约束规则，可被 AI 按需调用：
- 定义编码规范和风格要求
- 规定特定场景下的行为准则
- 设置禁止行为和边界条件

### 两者的关系

```
┌─────────────────────────────────────────┐
│           AI Coding Assistant           │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │     Rules (按需调用)             │   │
│  │  - 代码风格规则                  │   │
│  │  - 输出格式规则                  │   │
│  │  - 行为边界规则                  │   │
│  └─────────────────────────────────┘   │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │     Skills (按需激活)            │   │
│  │  ┌────────┐ ┌────────┐         │   │
│  │  │ Skill1 │ │ Skill2 │  ...    │   │
│  │  └────────┘ └────────┘         │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

## 核心区别

| 维度 | Skills | Rules |
|------|--------|-------|
| **定位** | 能力模块 | 行为约束 |
| **激活方式** | 显式调用或条件触发 | AI 自动判断是否调用 |
| **复杂程度** | 多步骤执行流程 | 单条或多条约束指令 |
| **内容侧重** | 做什么、怎么做 | 遵守什么、禁止什么 |
| **典型场景** | 代码审查、迁移、重构 | 代码风格、命名规范、安全约束 |

## 实际例子

### Rule 示例
```markdown
# 代码风格规则
- 使用 TypeScript 严格模式
- 函数必须有 JSDoc 注释
- 变量命名使用 camelCase
```

### Skill 示例
```markdown
# 数据库迁移 Skill

## 触发条件
当用户请求创建数据库迁移时激活

## 执行步骤
1. 分析当前数据库 schema
2. 生成迁移文件
3. 创建回滚脚本
4. 更新文档

## 输出
- migration_xxx.sql
- rollback_xxx.sql
```
