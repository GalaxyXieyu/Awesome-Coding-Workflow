# Skills 基础概念

## 为什么需要 Skills

### 问题背景

在使用 AI 编程助手（Cursor、Windsurf、Claude Code 等）进行开发时，我们经常遇到：
- **重复工作** - 每次都要重复说明相同的偏好和规范
- **上下文丢失** - 切换会话后之前的约定全部丢失
- **一致性问题** - 不同时间得到的输出格式/风格不一致
- **效率低下** - 大量时间花在"教育" AI 而非解决问题

### Skills 解决什么

**Skills** 是可复用的提示词模块，让 AI 能够：
- 执行特定领域的专业任务
- 遵循固定的执行流程
- 输出标准化的结果格式
- 遵守特定的行为约束和规范

### 工作原理

```
┌─────────────────────────────────────────┐
│           AI Coding Assistant           │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │         Skills (按需激活)          │   │
│  │  ┌────────┐ ┌────────┐         │   │
│  │  │ Skill1 │ │ Skill2 │  ...    │   │
│  │  └────────┘ └────────┘         │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

## Skills 的常见类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **流程型** | 定义多步骤执行过程 | 代码审查、数据库迁移、重构 |
| **约束型** | 定义行为规范和禁止项 | 代码风格、命名规范、安全约束 |
| **输出型** | 定义结果格式和模板 | 文档生成、报告输出、日志格式 |

## 示例

### 流程型 Skill
```markdown
# 数据库迁移 Skill

## 触发条件
当用户请求创建数据库迁移时激活

## 执行步骤
1. 分析当前数据库 schema
2. 生成迁移文件
3. 创建回滚脚本
4. 更新文档

## 输出
- migration_xxx.sql
- rollback_xxx.sql
```
