# Skill 快速模板

## 1. 流程型 Skill 模板

```markdown
---
name: {动词-名词}
description: {动作} + {对象} + {期望结果}
---

# {Skill 名称}

## 触发场景
- 用户说"{触发关键词}"

## 执行步骤
1. **{阶段1}**：{具体动作}
2. **{阶段2}**：{具体动作}
3. **{阶段3}**：{具体动作}

## 约束条件
- {必须遵守}
- {禁止行为}

## 输出格式
{标准化输出模板}
```

---

## 2. 约束型 Skill 模板

```markdown
---
name: {领域}-rules
description: {领域}开发规范和约束
---

# {领域}规范

## 适用场景
- 开发 {领域} 相关代码时

## 必须遵守
- {规则1}
- {规则2}

## 禁止行为
- {禁止1}
- {禁止2}

## 代码风格
- {风格要求}
```

---

## 3. 输出型 Skill 模板

```markdown
---
name: {输出物}-generator
description: 生成标准化的 {输出物}
---

# {输出物}生成器

## 触发场景
- 用户需要生成 {输出物}

## 输入要求
- {需要的上下文}

## 输出模板
\`\`\`
{模板内容}
\`\`\`

## 质量标准
- {标准1}
- {标准2}
```

---

## 精简原则

| 组件 | 目标词数 | 说明 |
|------|----------|------|
| description | <20 词 | 一句话概括 |
| 触发场景 | <50 词 | 明确触发条件 |
| 执行步骤 | <150 词 | 3-5 步为佳 |
| 约束条件 | <50 词 | 3-5 条为佳 |
| 输出格式 | <100 词 | 提供清晰模板 |

**总正文**：控制在 300 词以内

---

## 命名规范

| 类型 | 命名模式 | 示例 |
|------|----------|------|
| 流程型 | `{动作}-{对象}` | `code-review`, `data-explore` |
| 约束型 | `{领域}-rules` | `react-rules`, `sql-rules` |
| 输出型 | `{输出物}-gen` | `doc-gen`, `test-gen` |
