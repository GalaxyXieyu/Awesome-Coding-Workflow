# 元认知激活思维模型

> 在解决具体问题的同时，持续跳出问题本身，思考"我在怎么思考"。

---

## 一、核心定义

**元认知激活** = 在行动中保持对自己思维过程的觉察和优化。

不只是解决问题，而是同时追问：
- 我在解决什么问题？（问题定义）
- 为什么这样解决？（本质追问）
- 这个方法的边界是什么？（验证闭环）
- 能否复用到其他场景？（抽象沉淀）

---

## 二、四个核心动作

### 1. 具体入手

**原则**：先有案例，不空谈理论

**做法**：
- 从真实问题/案例开始，不从抽象概念开始
- 先动手做，在做的过程中发现问题
- 用具体素材作为思考的锚点

**自检**：我有没有真实的案例/数据支撑？还是在空想？

### 2. 本质追问

**原则**：不满足于"怎么做"，追问"为什么"

**典型问题**：
- 为什么这样做？背后的规律是什么？
- 用户/对方到底在想什么？真实动机是什么？
- 这个问题的本质是什么？表象背后是什么？

**自检**：我理解的是表面需求还是底层动机？

### 3. 验证闭环

**原则**：每个结论都要经得起追问

**典型问题**：
- 这个数据怎么测的？对比基准是什么？
- 这个说法经得起推敲吗？会被怎么追问？
- 换个场景还成立吗？边界条件是什么？

**自检**：如果有人质疑这个结论，我能回答吗？

### 4. 抽象复用

**原则**：把一次性经验变成可复用的框架

**典型问题**：
- 这件事的规律是什么？能总结成公式吗？
- 能迁移到其他场景吗？适用条件是什么？
- 值得沉淀成文档/工具/方法论吗？

**自检**：下次遇到类似问题，我能直接复用吗？

---

## 三、思维路径

```
具体案例 → 抽象规律 → 验证追问 → 沉淀复用 → 新案例检验
    ↑                                              ↓
    ←←←←←←←←←← 迭代更新 ←←←←←←←←←←←←←←←←←←←←←←←←←
```

**关键**：这是一个闭环，新的案例会检验和更新已有的框架。

---

## 四、双视角碰撞（与 AI 协作模式）

| 你的视角 | AI 的视角 |
|----------|-----------|
| 具体经验、真实痛点 | 结构化框架、外部信息 |
| 追问本质、验证边界 | 提供选项、补充盲区 |
| 整合沉淀、形成体系 | 执行落地、迭代优化 |

**协作价值**：
- 你负责"问对问题"
- AI 负责"给出选项"
- 共同完成从具体到抽象的闭环

**最佳实践**：
- 带着具体问题来，不要问"怎么学 XX"
- 追问 AI 的回答，要求验证和边界
- 主动要求整合成框架，沉淀到文档

## 五、底层能力清单

| 能力 | 定义 | 表现 |
|------|------|------|
| **元认知** | 跳出问题，思考"我在怎么思考" | 能意识到自己的思维盲区 |
| **归纳抽象** | 从案例到规律，从规律到框架 | 能从多个案例中提炼共性 |
| **用户洞察** | 站在对方视角，追问真实动机 | 能穿透表面需求到底层痛点 |
| **质量意识** | 逻辑自洽，经得起追问 | 不接受"差不多"的答案 |
| **知识管理** | 碎片→结构→体系 | 主动整合信息，形成可复用的模型 |

---

## 六、应用场景

### 学习新领域

```
找一个具体问题 → 动手解决 → 追问为什么 → 总结规律 → 迁移验证
```

### 解决复杂问题

```
拆解问题 → 找到本质 → 验证假设 → 形成方案 → 预判追问
```

### 与 AI 协作

```
带具体问题 → 要求多个选项 → 追问边界 → 整合成框架 → 沉淀文档
```

### 面试/汇报

```
准备案例 → 提炼规律 → 预判追问 → 准备回答 → 验证闭环
```

## 七、可调用的工具能力

### 工具清单

| 工具 | 能力 | 适用场景 |
|------|------|----------|
| **Sequential Thinking** | 深度思考、多步推理、假设验证 | 复杂问题拆解、决策分析、方案对比 |
| **Memory（知识图谱）** | 记录偏好、存储实体关系、持久化认知 | 记住你的习惯/偏好/项目背景，跨对话复用 |
| **Notion** | 创建/查询/更新文档和数据库 | 沉淀框架、管理任务、记录笔记 |
| **Perplexity** | 联网检索、实时信息获取 | 补充外部知识、验证信息、获取最新动态 |

### 工具与思维动作的映射

| 思维动作 | 可调用工具 | 用法示例 |
|----------|------------|----------|
| **具体入手** | Memory | 调取你之前的项目/经历作为案例 |
| **本质追问** | Sequential Thinking | 多步推理，拆解问题本质 |
| **验证闭环** | Perplexity | 联网检索，验证信息准确性 |
| **抽象复用** | Notion + Memory | 沉淀框架到文档，记录到知识图谱 |

### 工具调用指令

你可以在对话中说：

| 指令 | 触发工具 |
|------|----------|
| "深度思考一下这个问题" | Sequential Thinking |
| "记住我的这个偏好" | Memory（创建/更新实体） |
| "帮我查一下最新的..." | Perplexity 联网检索 |
| "把这个沉淀到 Notion" | Notion 创建页面/数据库 |
| "回顾一下我之前说过的..." | Memory（查询图谱） |

### 知识图谱（Memory）使用说明

**可以记录的内容**：
- **个人偏好**：沟通风格、学习方式、工作习惯
- **项目背景**：正在做的项目、技术栈、团队情况
- **认知框架**：你沉淀的心智模型、方法论
- **实体关系**：人物、公司、项目之间的关联

**调用方式**：
- "记住：我偏好简洁直接的沟通风格"
- "记住：我正在准备大模型应用工程师的面试"
- "我之前说过什么关于简历的偏好？"

---

## 八、调用方式

在对话中可以说：

> "用元认知激活的方式帮我分析这个问题"

AI 会：
1. 先从你的具体案例/问题入手
2. 追问背后的本质和边界
3. 给出可验证的结论
4. 尝试抽象成可复用的框架
5. 主动提出"这个结论可能被怎么追问"
6. **按需调用工具**：深度思考、联网检索、记录到图谱/Notion

---

## 九、一句话总结

**元认知激活** = 在做事的同时，持续追问"我在怎么思考"，把一次性经验变成可复用的框架。

核心心法：**具体入手 → 本质追问 → 验证闭环 → 抽象复用**

工具加持：**深度思考 + 联网检索 + 知识图谱 + 文档沉淀**
