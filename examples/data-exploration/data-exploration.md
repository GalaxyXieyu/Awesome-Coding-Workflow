# 数据探索 Skill

## 触发条件

### 显式触发
- 命令：`/explore-data`
- 请求："探索这个数据集"、"分析数据"

### 自动触发
- 检测到 CSV/Excel/JSON 数据文件
- 用户提到"数据探索"、"数据分析"

## 上下文要求

### 必需
- 数据源（文件路径或数据内容）
- 数据格式（CSV/JSON/Excel/SQL）

### 可选
- 分析目标描述
- 重点关注字段
- 数据字典/字段说明

## 执行步骤

### Step 1: 数据加载
1. 识别数据格式
2. 读取数据
3. 处理编码问题
4. 初步数据预览

### Step 2: 基础信息
1. 数据维度（行数、列数）
2. 字段列表及类型推断
3. 内存占用估算

### Step 3: 字段分析
对每个字段执行：

**数值型字段**：
- 统计量：均值、中位数、标准差、最小值、最大值
- 分布特征：偏度、峰度
- 缺失率

**类别型字段**：
- 唯一值数量
- 频次分布（Top 10）
- 缺失率

**日期型字段**：
- 时间范围
- 时间粒度推断
- 缺失率

### Step 4: 数据质量
1. 缺失值统计
2. 重复行检查
3. 异常值检测
4. 数据一致性检查

### Step 5: 生成报告
按规范格式输出分析报告

## 约束条件

### 能力边界
- 最大支持 100MB 数据文件
- 不处理图片/视频等非结构化数据
- 不执行修改数据的操作

### 安全约束
- 不在报告中展示敏感数据原值
- 示例数据需脱敏处理
- 不记录/传输原始数据

### 质量要求
- 所有统计数据可验证
- 明确标注推断和假设
- 提供数据依据

```
请分析 sales_data.csv 这个数据集
```
